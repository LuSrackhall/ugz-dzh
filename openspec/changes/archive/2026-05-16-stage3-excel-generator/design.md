## Context

项目已具备 voucher 解析（Stage 1）和 balance 余额管理（Stage 2）。balance 包提供 `LeafSummary`、`ComputeSummariesWithParents`、`Activity` 等核心类型。Stage 3 需要基于这些输入生成本月完整累计 xlsx 工作薄。目标平台为 Go 1.21+，依赖 `excelize/v2`。

约束：
- 历史 xlsx 文件永不修改
- 每月 xlsx 完整独立（包含从启动月至当月的所有科目）
- 金额内部使用 int64 分，显示为 float64 元
- 科目全路径 = 总账科目 + "-" + 明细科目

## Goals / Non-Goals

**Goals:**
- 为每个叶子科目生成独立的总分类账 Sheet
- 为有明细的总账科目生成多科目明细账 Sheet（基础列 + 扩展列）
- 多科目明细账中按总账科目汇总其下所有明细
- 每页固定 20 行数据行 + 过次页行
- 自动处理期初结转、月结、余额计算
- 打印标记：有变化的数据行标记为需打印
- 生成流程：读取上月 xlsx → 提取期末 → 生成本月期初表 → 追加分录 → 月结

**Non-Goals:**
- 不处理跨年结转（Stage 5）
- 不生成多科目明细账配置的 JSON 编辑 UI
- 不实现 ledger generate 之外的 CLI 子命令（Stage 4）

## Decisions

### 1. 数据结构：Workbook 为顶层类型

Workbook 持有 excelize.File、配置引用、当月标识，所有 Sheet 操作通过 Workbook 方法完成。

### 2. Sheet 命名规则

- 总分类账 Sheet: `总分类账-{科目全路径}`（如 `总分类账-库存现金`、`总分类账-管理费用-办公费`）
- 多科目明细账 Sheet: `多科目明细账-{总账科目}`（如 `多科目明细账-管理费用`）
- 期初表 Sheet: `{YYYY-MM}期初`

### 3. 分页逻辑

每页结构：标题行(1) → 列标题行(2) → 数据行×20(3-22) → 过次页行(23)。满 20 行数据后自动开新页，页码连续。过次页行的余额结转至下页首行。

### 4. 余额计算

- 首数据行：余额 = 期初(或上页过次页余额) + 借方 - 贷方
- 后续行：余额 = 上行余额 + 借方 - 贷方
- 过次页行：余额 = 最后数据行的余额

### 5. 多科目明细账扩展列

列 H-U（14列）对应该总账科目下的各明细科目。A-G 列同总分类账（日期/凭证号/摘要/借方/贷方/方向/余额），H-U 列每列一个明细科目，填入该笔分录在该明细下的金额。父级汇总行显示各明细列的合计。

### 6. 打印标记

- 当期月有数据写入的行标记为"需打印"
- 跨月未满页时，旧月数据行也标记为"需打印"
- 标记存储在隐藏列或通过 cell comment 实现

### 7. 生成流程（month 参数为 YYYY-MM）

1. `GenerateWorkbook(configPath, voucherDir, month, outputDir)` — 入口
2. 加载 balance config（`科目余额总览.json`）
3. 解析当月凭证 → `[]Entry`
4. 若存在上月 xlsx，复制之；否则新建空白工作薄
5. 从上月 xlsx 提取各 Sheet 过次页行的期末余额（即本月期初）
6. 计算本月各科目活动量（`map[string]Activity`）
7. 生成本月期初表 Sheet
8. 对每个叶子科目，追加分录到其总分类账 Sheet（自动分页）
9. 对有明细的总账科目，追加分录到其多科目明细账 Sheet
10. 执行月末结账（本月合计、本年累计行）
11. 标记需打印的页
12. 回写 balance 余额历史（`UpdateBalancesAfterGenerate`）
13. 保存 xlsx

## Risks / Trade-offs

- **excelize 性能**: 大量 Sheet（可能数十个科目）和格式设置可能导致生成缓慢 → 批量设置列宽和样式，按需创建 Sheet
- **上月 xlsx 缺失**: 首次生成时无上月文件 → 新建空工作薄，期初从 JSON 的 firstRecord 获取
- **科目增删**: 凭证中出现新科目需自动创建 Sheet → 在追加分录阶段检测并创建
