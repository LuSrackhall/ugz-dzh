# 手工账电子化生成系统

## 系统目标

构建**手工账电子化生成系统**。输入按月整理的凭证 Markdown 文件（内含 HTML 表格分录）和一份全局 JSON 配置文件，自动生成每月独立、完整的累计 Excel 工作薄（`.xlsx`）。Excel 中按末端科目分 Sheet，支持**总分类账**和**多科目明细账**两种账页格式，每页固定 20 行数据行加"过次页"行，自动处理期初结转、月结、余额计算、打印标记，适配活页打印与 git 版本追踪。

## 核心数据流

```
凭证 .md 文件 ──→ voucher 解析器（封装现有逻辑）──→ []Entry
                              │
全局 JSON (科目余额总览.json) ──→ balance 管理器 ──→ 当月期初 & 更新余额历史
                              │
上月 .xlsx ────────────────────→ generator 生成器 ──→ 本月 .xlsx（独立完整）
```

- **期初权威来源**：JSON 中的"自动识别科目.期初调整额"或"手动调整科目.期初调整额"，结合上月 xlsx 的期末余额动态计算。
- **历史安全铁律**：每月生成时完整复制上月 xlsx 再追加当月数据，历史 xlsx 文件永不修改。
- **Git 可观测**：所有科目变动、余额变化均反映在 `科目余额总览.json` 的纯文本 diff 中。

## 技术栈与文件结构

- Go 1.21+，`excelize/v2`，标准库
- 目录结构：

```

```

## 不可动摇的设计铁律

1. **零手改 xlsx 期初表**：xlsx 中的 `YYYY-MM期初` 工作表只是生成品，可由脚本自动重建。期初调整的唯一入口是 JSON 中的 `期初调整额` 字段。
2. **每月 xlsx 完整独立**：例如 `output/2026-08.xlsx` 包含从启动月至 8 月的所有科目完整明细，页码连续，可独立打印。
3. **科目自动管理**：凭证中新出现的科目自动加入 JSON 的 `自动识别科目` 列表，期初默认 0。用户可在 JSON 中修改其 `期初调整额`。
4. **凭证解析不改逻辑**：完全保留现有的 HTML 表格解析代码，凭证格式已固定，系统无需关心内容设计。
5. **打印友好**：每月仅标记有变化的账页为"需打印"，用户按标记打印后替换活页即可。

## 关键约定

- 科目全路径 = `总账科目 + "-" + 明细科目`（若明细科目为空，则只用总账科目）。
- 所有内部金额使用 `int64` 分，JSON 中存储为 `float64` 元，显示时转为元并保留两位小数。
- 科目属性（借/贷）在 JSON 的科目树节点中存储，可由用户手动修改，默认自动推断。

### 阶段 2：全局 JSON 余额管理器（balance 包）

实现 `科目余额总览.json` 的加载、保存，以及核心的期初计算、余额回写、科目前推逻辑。

JSON 结构：
- `全局设置`：启动月、科目顺序
- `科目树`：每个科目（叶子路径）的属性、首次记录、按月余额历史
- `自动识别科目`：从凭证中自动发现的科目及期初调整额
- `手动调整科目`：用户手动添加的科目及生效月、调整额、说明

主要函数：
- `LoadConfig / SaveConfig` — 读写 JSON，确保 git diff 友好
- `UpdateBalancesAfterGenerate` — 每月生成后回写余额
- `GetInitBalanceForGenerate` — 获取某月期初的核心函数
- `AddManualAdjustment` — 添加手动调整科目
- `GetLeafAccounts` — 返回所有叶子科目全路径
- `ValidateAccountTree` — 验证科目树完整性

关键逻辑：
- 期初计算优先级：上月期末 > 首次记录调整额 > 0
- 期初前推：调整额 ≠ 0 的科目，从启动月到首次记录月-1 补齐余额记录
- 科目自动创建：凭证中新科目自动加入科目树

### 阶段 3：Excel 累计工作薄生成器（generator 包）

基于 voucher、balance 和上月 xlsx，生成本月新的累计 xlsx。

**总分类账 Sheet**（每个叶子科目一张）：
- 标题行 → 列标题（日期/凭证号/摘要/借方/贷方/方向/余额）→ 20行数据 → 过次页行
- 余额公式：页首行引用上页过次页余额或期初；后续行 = 上行余额 + 借方 - 贷方
- 期初处理：新出现且期初 ≠ 0 的科目，首行插入"上年结转"

**多科目明细账 Sheet**：
- 基础列 A-G 同总分类账
- 扩展列 H-U 对应各明细科目
- 配置来源：JSON 中 `多科目明细账配置`

**生成流程**：
1. 复制上月 xlsx（或新建）
2. 提取上月期末数据（各 Sheet 最后过次页行的 G 列值）
3. 生成本月期初表（`YYYY-MM期初` Sheet）
4. 逐笔追加分录（自动分页、填满后开新页）
5. 月末结账（本月合计、本年累计行）
6. 标记需打印的页

**打印标记**：
- 所有新写入数据行标记为"需打印"
- 跨月未满页时，该页原有数据行也标记为"需打印"

### 阶段 4：命令行集成与辅助工具（main.go）

子命令：
- `ledger generate` — 生成月度账本（-month, -voucherDir, -json, -outputDir, -force, -verbose）
- `ledger check` — 检测 JSON 科目树与余额
- `ledger reset` — 重置打印标记
- `ledger add-manual` — 手动添加调整科目
- `ledger init` — 系统初始化（创建初始 JSON）
- `ledger year-close` — 跨年结转
